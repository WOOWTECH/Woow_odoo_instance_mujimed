## 2. 會員登入與註冊

### 步驟 1：登入頁面

![登入頁面：輸入帳號與密碼後點擊登入按鈕](images/ch02_01_login_page.png)

1. 開啟瀏覽器，前往 `https://inzense-odoo.woowtech.io/web/login`。
2. 頁面顯示登入表單，包含電郵與密碼欄位。

| 功能 | 說明 |
|------|------|
| 電郵欄位 | 輸入登入帳號（Email 或使用者名稱） |
| 密碼欄位 | 輸入對應密碼 |
| 登入按鈕 | 點擊後驗證帳號密碼 |
| 忘記密碼 | 點擊後前往密碼重設頁面 |
| 註冊連結 | 點擊後前往新會員註冊頁面 |

### 步驟 2：會員登入

![填入 Portal 會員帳號密碼，準備登入](images/ch02_02_login_filled.png)

1. 在「電郵」欄位輸入 `portal@inzense.com.tw`。
2. 在「密碼」欄位輸入 `portal`。
3. 點擊「登入」按鈕。
4. 登入成功後，自動跳轉至 Portal 首頁（`/my/home`）。

![Portal 會員登入後看到的首頁畫面](images/ch02_05_portal_after_login.png)

### 步驟 3：管理員登入

1. 在「電郵」欄位輸入 `admin`。
2. 在「密碼」欄位輸入 `admin`。
3. 登入成功後，進入後台管理介面。

![管理員登入後進入 Portal 首頁](images/ch02_04_admin_after_login.png)

### 登入錯誤處理

![輸入錯誤密碼時，系統顯示「登入名稱或密碼錯誤」提示](images/ch02_03_login_error.png)

當輸入的帳號或密碼不正確時，系統會在登入表單上方顯示錯誤提示訊息。請確認帳號與密碼是否正確後重新嘗試。

### 步驟 4：忘記密碼

1. 在登入頁面點擊「重設新密碼」連結。
2. 輸入註冊時使用的電子郵件地址。
3. 點擊「送出」按鈕。
4. 系統發送密碼重設郵件至您的信箱。
5. 開啟郵件，點擊重設連結，設定新密碼。

### 步驟 5：新會員註冊

1. 在登入頁面點擊「還沒有帳戶？」連結。
2. 填寫姓名、電子郵件、密碼等資訊。
3. 點擊「註冊」按鈕完成註冊。

> **安全提醒**：預設的 `admin` / `admin` 帳號密碼僅供展示與測試使用，正式上線前務必修改為強密碼。

## 3. 會員中心（Portal 儀表板）

登入後，會員將進入 Portal 儀表板頁面。此頁面為會員的個人中心，提供快速導覽至各項功能的入口。

### 步驟 1：儀表板總覽

![Portal 儀表板顯示歡迎訊息、近期通知與功能入口](images/ch03_01_portal_dashboard.png)

Portal 儀表板顯示會員的各項服務統計與快速入口：

| 功能 | 說明 |
|------|------|
| 歡迎訊息 | 顯示個人化問候語與日期 |
| 你的訂單 | 顯示訂單數量，點擊前往訂單列表 |
| 發票 | 顯示發票數量，點擊前往發票列表 |
| 會員中心 | 點擊前往忠誠計畫總覽（集點卡、優惠券、禮品卡、電子錢包） |

### 步驟 2：功能卡片與路徑

![Portal 功能卡片區塊，包含訂單、發票、會員中心等入口](images/ch03_02_portal_cards.png)

| 功能卡片 | 路徑 | 說明 |
|---------|------|------|
| 你的訂單 | `/my/orders` | 查看所有銷售訂單 |
| 發票 | `/my/invoices` | 查看所有發票與付款紀錄 |
| 會員中心 | `/my/member-center` | 查看忠誠計畫（集點、優惠、禮品、錢包） |
| 個人資料 | `/my/account` | 編輯個人資料與聯絡資訊 |
| 連線及保安 | `/my/security` | 修改密碼與安全設定 |

![個人資料編輯頁面，可修改姓名、電話、地址等資訊](images/ch03_03_portal_account.png)

### 步驟 3：Portal 首頁完整畫面

![Portal 首頁完整畫面](images/ch03_04_portal_home_full.png)

Portal 首頁提供會員完整的功能總覽。畫面上方顯示歡迎訊息與個人化問候語，中間區域排列各功能模組的入口卡片。
會員可以在此頁面一目瞭然地掌握訂單數量、發票數量、以及忠誠計畫的各項餘額。
首頁採用響應式設計，在桌面版與行動裝置上均能正常顯示。

### 步驟 4：功能卡片詳細說明

![Portal 功能卡片詳細](images/ch03_05_portal_cards_detail.png)

Portal 功能卡片是會員最常使用的導覽入口。每張卡片顯示對應功能的摘要資訊：
- **訂單卡片**：顯示目前的訂單總數，點擊可直接進入訂單列表頁面。
- **發票卡片**：顯示發票數量與付款狀態摘要，方便會員追蹤款項。
- **會員中心卡片**：顯示忠誠計畫的概覽資訊，包含集點卡、優惠券、禮品卡與電子錢包。
- **個人資料卡片**：可快速進入個人資料編輯頁面。

### 步驟 5：安全性與密碼修改

![Portal 安全性/密碼修改頁面](images/ch03_06_portal_security.png)

在 Portal 中點擊「連線及保安」或前往 `/my/security`，即可進入安全性設定頁面：

1. 在「目前密碼」欄位輸入您現有的密碼。
2. 在「新密碼」欄位輸入要設定的新密碼。
3. 在「確認新密碼」欄位再次輸入新密碼以確認。
4. 點擊「變更密碼」按鈕完成修改。

> **安全建議**：建議定期更換密碼，並使用包含大小寫英文、數字與特殊符號的強密碼，以確保帳號安全。

> **提示**：展示客戶帳號已預設有 10+ 筆訂單、4 種忠誠計畫卡片，可完整體驗所有 Portal 功能。

### 步驟 1：進入付款畫面

1. 確認購物車中的商品正確後，點擊「付款」按鈕。
2. 系統進入付款畫面，顯示應付金額與可用的付款方式。

收銀人員可依據客戶的付款需求，點擊對應的付款方式。系統支援單一付款與混合付款兩種模式。

### 步驟 2：現金付款

1. 在付款畫面點擊「現金」付款方式（如「現金-信義」）。
2. 系統自動帶入應付金額。
3. 如客戶支付金額大於應付金額，輸入實收金額，系統自動計算找零。
4. 點擊「確認」按鈕完成交易。
5. 列印收據或發送電子收據。

![POS 現金付款已選擇，顯示金額與確認按鈕](images/ch11_02_pos_cash_selected.png)

![POS 現金付款已選](images/ch11_06_pos_cash_selected.png)

選擇現金付款後，系統自動帶入應付金額。畫面顯示應付金額、已收金額與找零金額。
若客戶支付的金額大於應付金額，收銀人員可在數字鍵盤上輸入實際收到的金額，系統會自動計算找零。
確認金額無誤後，點擊「確認」按鈕即可完成交易。

![POS 交易完成收據畫面](images/ch11_03_pos_receipt.png)

![POS 收據畫面](images/ch11_09_pos_receipt.png)

交易完成後，系統自動顯示收據畫面。收據包含門市名稱、交易日期時間、購買商品明細、各項金額與付款方式。
收銀人員可選擇列印紙本收據或發送電子收據至客戶的電子郵件。點擊「列印收據」按鈕即可透過連接的收據印表機列印。

![POS 新訂單畫面，購物車已清空準備下一筆交易](images/ch11_04_pos_new_order.png)

![POS 新訂單準備](images/ch11_10_pos_new_order.png)

完成一筆交易後，POS 自動切換至新訂單畫面。購物車已清空，準備接待下一位客戶。
收銀人員可立即開始選擇商品，進行下一筆交易。畫面上方的訂單編號會自動遞增。

### 步驟 3：信用卡付款

![POS 信用卡付款](images/ch11_07_pos_credit_card.png)

1. 在付款畫面點擊「禪香不二 信用卡」付款方式。
2. 系統帶入應付金額，畫面顯示信用卡付款已選取。
3. 使用實體刷卡機完成刷卡（Odoo 不處理實際刷卡，僅記錄金額）。
4. 確認刷卡成功後，點擊「確認」按鈕完成交易。

> **注意**：信用卡付款方式在 Odoo 中僅記錄金額，實際的刷卡授權需透過外接的刷卡機完成。

### 步驟 4：客戶帳戶付款（先出貨後收款）

![POS 客戶帳戶付款](images/ch11_08_pos_customer_account.png)

此付款方式用於外部平台訂單或需要延遲收款的場景。選擇「客戶帳戶」後，畫面顯示該付款方式已選取，金額自動帶入。

1. 在付款畫面點擊「客戶帳戶」付款方式。
2. 必須先指定客戶（客戶帳戶需要記錄應收帳款對象）。
3. 系統將訂單記錄為「應收帳款」，不立即產生現金/銀行分錄。
4. 等外部平台撥款或客戶付款後，在後台進行銷帳。

| 付款方式 | 類型 | 即時入帳 | 適用場景 |
|---------|------|---------|---------|
| 現金 | cash | 是 | 門市現場現金交易 |
| 信用卡 | bank | 是 | 門市現場刷卡交易 |
| 客戶帳戶 | pay_later | 否（應收） | 外部平台訂單、延遲收款 |

### 步驟 5：混合付款

POS 支援單筆訂單使用多種付款方式：

1. 先點擊第一種付款方式（如現金），輸入部分金額。
2. 再點擊第二種付款方式（如信用卡），系統自動帶入剩餘金額。
3. 確認總付款金額等於應付金額後，點擊「確認」完成交易。

## 12. POS 銷售點 — 會員優惠折扣

POS 整合了 4 種忠誠計畫，會員在 POS 結帳時可使用各種優惠折扣。

### 步驟 1：集點卡折抵

1. 先指定客戶（必須）。
2. 系統自動查詢該客戶的集點卡餘額。
3. 結帳時，系統會顯示可用的集點折扣選項。
4. 選擇使用集點折抵，輸入要折抵的點數。
5. 折抵規則：1 點 = NT$1。

**範例**：客戶有 210 點，購買巴拉圭綠檀 NT$1,000，使用 100 點折抵，實付 NT$900。

### 步驟 2：優惠券使用

1. 先指定客戶。
2. 在 POS 中輸入優惠券代碼。
3. 系統驗證代碼是否有效：
   - 訂單金額是否滿足最低消費（NT$1,000）。
   - 優惠券是否已使用。
4. 驗證通過後，自動折抵 NT$100。

### 步驟 3：禮品卡折抵

1. 先指定客戶。
2. 系統查詢該客戶持有的禮品卡及餘額。
3. 結帳時選擇禮品卡折抵。
4. 折抵金額不超過禮品卡餘額與訂單金額的較小值。

**範例**：GIFT-A-500 餘額 NT$500，購買木神香 NT$750，折抵 NT$500，實付 NT$250。

### 步驟 4：電子錢包折抵

1. 先指定客戶。
2. 系統顯示電子錢包餘額。
3. 結帳時選擇電子錢包折抵，輸入折抵金額。
4. 折抵規則：1 點 = NT$1。

**範例**：電子錢包餘額 NT$3,000，購買善緣香 NT$899，折抵 NT$500，實付 NT$399。

2. 點擊「關閉收銀機」或「Close」。
3. 系統進入關帳流程。

![POS 漢堡選單，提供關閉收銀機與其他設定選項](images/ch15_01_pos_hamburger_menu.png)

![POS 開啟收銀機對話框](images/ch15_03_pos_opening_dialog.png)

開啟收銀機時，系統會顯示開店對話框。收銀人員可在此輸入期初現金金額（即收銀機中的初始現金）。
輸入正確的期初金額有助於日結時核對現金收入是否正確。設定完成後點擊「開啟」即可開始營業。

### 步驟 2：日結摘要

關閉收銀機時，系統顯示日結摘要：

| 項目 | 說明 |
|------|------|
| 交易筆數 | 本次 session 的總交易筆數 |
| 總營業額 | 所有交易的總金額 |
| 現金收入 | 現金付款的總金額 |
| 信用卡收入 | 信用卡付款的總金額 |
| 客戶帳戶 | 延遲收款的金額（應收帳款） |

![POS 關店日結摘要](images/ch15_04_pos_closing_summary.png)

日結摘要畫面詳細列出本次營業時段的所有交易統計。收銀人員應仔細核對各項金額：
- 確認現金收入金額與收銀機中的實際現金一致。
- 確認信用卡收入與刷卡機的交易紀錄吻合。
- 確認客戶帳戶的應收金額正確。

### 步驟 3：確認關帳

1. 核對日結摘要數據是否正確。
2. 點擊「關閉」或「驗證」按鈕確認關帳。
3. 系統將 POS session 狀態設為「已關閉」。
4. 庫存扣除與會計分錄在此時生成。

![POS 關店完成](images/ch15_05_pos_closed.png)

關帳完成後，系統顯示確認訊息，POS session 狀態變為「已關閉」。
此時系統已自動處理所有庫存扣除與會計分錄的記帳。管理員可在後台查看該 session 的完整交易報表。

> **注意**：POS session 關閉後，該 session 內的所有訂單即為最終確認，無法再修改。

## 16. POS 訂單批量匯入

本功能用於將外部電商平台（如蝦皮、momo）的訂單批量匯入至 POS 系統。匯入的訂單使用「客戶帳戶」（pay_later）付款方式，記錄為應收帳款。

### 步驟 1：開啟匯入精靈

![POS 模組首頁](images/ch16_03_pos_module_home.png)

1. 以管理員帳號登入後台。
2. 進入「銷售點」模組，首頁顯示所有門市的 POS 設定卡片與快速操作入口。
3. 點擊上方選單列的「訂單」，進入 POS 訂單管理頁面。

![POS 訂單頁面](images/ch16_04_pos_orders_page.png)

POS 訂單頁面列出所有已建立的 POS 訂單。頁面上方提供「匯入 POS 訂單」的功能入口。
點擊「匯入 POS 訂單」按鈕，系統彈出匯入精靈對話框。

### 步驟 2：匯入精靈欄位說明

![POS 訂單匯入精靈，選擇門市與上傳檔案](images/ch16_01_pos_import_wizard.png)

| 欄位 | 說明 |
|------|------|
| 出貨門市 | 選擇目標 POS 門市（必填，決定從哪個倉庫扣庫存） |
| 檔案類型 | CSV 或 Excel (.xlsx) |
| 檔案 | 點擊「上載檔案」上傳訂單檔案 |
| 產品比對方式 | 名稱 / 內部參照 / 條碼（用於比對商品） |
| 錯誤處理 | 嚴格模式（全部或不匯入）/ 寬鬆模式（跳過錯誤行） |

### 步驟 3：CSV 檔案格式

| 欄位名稱 | 必填 | 說明 |
|---------|------|------|
| ORDER | 是 | 訂單編號（同一編號的行合併為一筆訂單） |
| CUSTOMER | 是 | 客戶名稱（須與系統中的客戶名稱完全一致） |
| PRODUCT | 是 | 商品名稱（或內部參照/條碼，依比對方式） |
| QUANTITY | 是 | 數量 |
| PRICE | 是 | 單價 |
| DATE | 是 | 日期（YYYY-MM-DD 或 DD/MM/YYYY） |
| TAX | 否 | 稅項名稱（多個以分號隔開） |
| DISCOUNT | 否 | 折扣百分比（如 5 代表 5%） |

**CSV 範例：**

```csv
ORDER,CUSTOMER,PRODUCT,QUANTITY,PRICE,DATE,TAX,DISCOUNT
SHOPEE-001,禪香不二 展示客戶,線香功能系列 – 善緣香,2,899,2026-05-20,,
SHOPEE-001,禪香不二 展示客戶,線香功能系列 – 財神香,1,899,2026-05-20,,
SHOPEE-002,禪香不二 展示客戶,柬埔寨沉香,1,1200,2026-05-21,,5
```

### 步驟 4：下載範本

![匯入精靈下方的範本下載按鈕](images/ch16_02_pos_import_download.png)

- 點擊「下載 CSV 範本」可下載預填範例的 CSV 檔案。
- 點擊「下載 Excel 範本」可下載預填範例的 Excel 檔案。
- 下載範本不需要先填寫任何欄位。

### 步驟 5：執行匯入

1. 選擇出貨門市（如「禪香不二 信義旗艦店」）。
2. 上傳 CSV 或 Excel 檔案。
3. 點擊「匯入」按鈕。
4. 系統自動：
   - 為該門市開啟一個批量匯入 session。
   - 逐筆建立 POS 訂單，付款方式為「客戶帳戶」。
   - 關閉 session，觸發庫存扣除。
5. 匯入完成後，顯示匯入結果與建立的訂單列表。

### 步驟 6：錯誤處理

| 模式 | 行為 |
|------|------|
| 嚴格模式 | 先驗證所有行，任何一行有錯誤則全部不匯入 |
| 寬鬆模式 | 匯入有效的行，跳過錯誤行，並在結果中列出被跳過的行 |

常見錯誤：
- 找不到客戶名稱
- 找不到商品名稱
- 數量或價格不是有效數字
- 缺少必填欄位

| 檔案 | 點擊「上載檔案」上傳訂單檔案 |
| 產品比對方式 | 名稱 / 內部參照 / 條碼 |
| 匯入狀態 | 草稿報價（僅建立）/ 自動確認（建立並確認） |
| 錯誤處理 | 嚴格模式 / 寬鬆模式 |

### 步驟 3：CSV 格式

CSV 格式與 POS 匯入相同（ORDER, CUSTOMER, PRODUCT, QUANTITY, PRICE, DATE, TAX, DISCOUNT）。

### 步驟 4：匯入狀態選項

| 選項 | 說明 |
|------|------|
| 草稿報價 | 匯入後建立為「報價單」（draft），需手動確認 |
| 自動確認 | 匯入後自動確認為「銷售訂單」（sale） |

### 步驟 5：執行匯入

1. 選擇檔案類型並上傳檔案。
2. 選擇產品比對方式與匯入狀態。
3. 點擊「匯入」按鈕。
4. 匯入完成後，顯示建立的銷售訂單列表。

## 18. 銷售訂單完整流程

本章說明銷售訂單從建立到完成收款的完整生命週期。

### 步驟 1：建立銷售訂單

![銷售訂單列表，顯示所有訂單的狀態與金額](images/ch18_01_so_list.png)

![銷售訂單列表含狀態](images/ch18_05_so_list_status.png)

銷售訂單列表以表格形式呈現所有訂單，每筆訂單清楚顯示訂單編號、客戶名稱、訂單日期、金額與目前狀態。
狀態欄以不同顏色標籤區分：「報價單」顯示為灰色、「銷售訂單」為藍色、「已完成」為綠色，方便管理員快速辨識訂單進度。

![銷售訂單詳情，上方顯示狀態按鈕與客戶資訊](images/ch18_02_so_detail.png)

![已確認銷售訂單](images/ch18_06_so_confirmed.png)

已確認的銷售訂單頁面上方顯示狀態流程列，清楚標示目前訂單處於「銷售訂單」階段。
頁面中間為客戶資訊、訂單日期與到期日等欄位。已確認的訂單可進行出貨與開票等後續操作。

1. 進入「銷售 > 訂單」。
2. 點擊「新增」建立新的銷售訂單。
3. 選擇客戶。
4. 在訂單明細中新增商品行：選擇商品、設定數量。
5. 系統自動計算小計、稅額、合計。
6. 點擊「儲存」保存草稿。

![銷售訂單明細行，顯示商品、數量、單價、小計](images/ch18_03_so_order_lines.png)

![新建銷售訂單的空白表單](images/ch18_04_so_create_new.png)

![新建銷售訂單表單](images/ch18_09_so_new_form.png)

新建銷售訂單表單提供完整的訂單建立介面。上方選擇客戶後，在「訂單行」區段點擊「新增一行」加入商品。
選擇商品後系統自動帶入該商品的銷售價格，收銀人員可依需要調整數量與折扣。
表單底部即時計算未稅小計、稅額與訂單總金額。確認內容無誤後點擊「儲存」保存為草稿。

### 步驟 2：確認訂單

1. 在銷售訂單頁面，點擊「確認」按鈕。
2. 訂單狀態從「報價單」變為「銷售訂單」。
3. 系統自動建立出貨單（stock.picking）。

### 步驟 3：處理出貨

1. 在已確認的訂單頁面，點擊「出貨」按鈕或進入出貨單。
2. 確認各商品的實際出貨數量。
3. 點擊「驗證」完成出貨。
4. 庫存從對應倉庫扣除。

### 步驟 4：建立發票

1. 在已出貨的訂單頁面，點擊「建立發票」按鈕。
2. 選擇發票類型（一般發票）。
3. 系統自動產生發票草稿。
4. 確認發票明細無誤。

### 步驟 5：過帳發票

1. 在發票頁面，點擊「確認」按鈕。
2. 發票狀態從「草稿」變為「已過帳」。
3. 系統產生會計分錄（journal entry）。
4. 發票編號正式生成（如 INV/2026/XXXXX）。

### 步驟 6：登記收款

1. 在已過帳的發票頁面，點擊「登記付款」按鈕。
2. 選擇付款方式（銀行/現金）。
3. 輸入收款金額（可全額或部分收款）。
4. 輸入付款日期。
5. 點擊「確認」完成收款。
6. 發票狀態變為「已付款」。

### 流程總結

```
報價單（Draft） → 銷售訂單（Sale） → 出貨（Done） → 發票草稿（Draft）
→ 已過帳（Posted） → 已付款（Paid）
```

## 27. 常見問題（FAQ）

### Q1：如何登入會員中心？

**A**：前往 `https://inzense-odoo.woowtech.io/web/login`，輸入帳號 `portal@inzense.com.tw`、密碼 `portal`，登入後從首頁點擊「會員中心」即可。

### Q2：如何查看集點卡餘額？

**A**：登入後前往 `/my/member-center`，在會員中心 Hub 頁面即可看到集點卡的目前點數。點擊可進入詳情查看歷史紀錄。

### Q3：如何使用優惠券？

**A**：優惠券可在 POS 結帳時使用。收銀人員在 POS 畫面輸入優惠券代碼即可套用折扣。使用條件：訂單金額需滿 NT$1,000。

### Q4：如何在 POS 使用電子錢包折抵？

**A**：在 POS 先指定客戶，系統自動查詢電子錢包餘額。在付款畫面選擇使用電子錢包折抵，輸入折抵金額後完成付款。

### Q5：如何開啟 POS 收銀機？

**A**：以管理員身份登入後台，進入「銷售點」模組，選擇門市後點擊「開啟收銀機」。

### Q6：三間門市的庫存是獨立的嗎？

**A**：是的。每間門市有專屬倉庫，POS 銷售僅從該門市倉庫扣庫存，不影響其他門市。

### Q7：門市庫存不足怎麼辦？

**A**：在後台「庫存 > 作業 > 調撥」中建立內部調撥，從中央倉庫調貨至門市倉庫。

### Q8：如何批量匯入外部平台訂單？

**A**：進入「銷售點 > 訂單 > 匯入 POS 訂單」，選擇門市、上傳 CSV/Excel 檔案，點擊「匯入」。系統自動建立 POS 訂單並從該門市倉庫扣庫存。

### Q9：POS 匯入的訂單使用什麼付款方式？

**A**：統一使用「客戶帳戶」（pay_later）付款方式。訂單記錄為應收帳款，等外部平台撥款後再在後台進行銷帳。

### Q10：如何銷帳 POS 的應收帳款？

**A**：在後台「會計」模組中找到對應的應收帳款分錄，使用「登記付款」功能記錄收款即可完成銷帳。

### Q11：如何批量匯入銷售訂單？

**A**：進入「銷售 > 訂單 > 匯入銷售訂單」，上傳 CSV/Excel 檔案，選擇匯入狀態（草稿或自動確認），點擊「匯入」。

### Q12：銷售訂單的完整流程是什麼？

**A**：建立報價單 → 確認為銷售訂單 → 處理出貨 → 建立發票 → 過帳發票 → 登記收款。

### Q13：如何開立發票？

**A**：在已確認的銷售訂單中點擊「建立發票」，選擇發票類型後確認。發票建立後需另外點擊「確認」進行過帳。

### Q14：如何處理退款？

**A**：在已過帳的發票中點擊「建立折讓單」（Credit Note），輸入退款原因與金額，確認後系統自動產生退款分錄。

### Q15：POS 可以開發票嗎？

**A**：POS 訂單預設不開發票。如需開立，可在後台「銷售點 > 訂單」中找到該筆 POS 訂單，手動建立發票。

### Q16：有哪些商品可以在 POS 銷售？

**A**：系統共有 10 項商品，分為功能系列（3 項）、脈輪系列（1 項）、五行系列（1 項）、拜拜用香（2 項）、優惠組合（3 項），全部可在 POS 中銷售。

### Q17：如何修改個人資料？

**A**：登入 Portal 後前往 `/my/account`，可修改姓名、電話、地址等資訊。

### Q18：如何修改密碼？

**A**：登入 Portal 後前往 `/my/security`，輸入目前密碼與新密碼即可修改。

### Q19：手機可以正常使用嗎？

**A**：是的。系統採用響應式設計，Portal 會員中心、後台管理均可在手機上正常操作。

### Q20：如何聯繫禪香不二？

**A**：電話：0926-926-851，地址：台北市北投區光明路240號2樓之8。
